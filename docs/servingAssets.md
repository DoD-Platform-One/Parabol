# Serving Assets

Parabol can serve assets in multiple ways, depending on the configuration through the variables `FILE_STORE_PROVIDER`, `CDN_BASE_URL` and `PROXY_CDN`.

## Types of assets

- **Static files:** build files of Parabol and some static images.
- **Dynamic assets:** files uploaded by users and generated by the application. This includes user profile pictures, uploaded files in pages and similar assets.

## Local Storage

> **This is the default configuration and it is not intended for production use.**

When `FILE_STORE_PROVIDER` is set to `local`, Parabol will serve assets from the local storage. 

- Static files live in the folder `/home/node/parabol/build` in the container.
- Dynamic assets live in the folder `/home/node/parabol/self-hosted` in the container.

When using local storage, you can configure the chart to use a persistent volume to store the dynamic assets and share that volume between the containers.

Example of configuration to create shared volume for local storage, automatically mounted in every Parabol pod:

```yaml
services:
  parabol:
    localStorage:
      enabled: true
      volumeSize: 1Gi
      awsEbs: false
      storageClassName: "local-path"
      accessModes:
        - ReadWriteOnce
parabolDeployment:
  env:
    fileStoreProvider: local
```

The chart can be configured to serve assets through Nginx instead of through NodeJS. This is an example using `local-path` storage class, but it can be used with any storage class (e.g. efs):

```yaml
services:
  parabol:
    localStorage:
      enabled: true
      volumeSize: 1Gi
      awsEbs: false
      storageClassName: "local-path"
      accessModes:
        - ReadWriteOnce
parabolDeployment:
  nginx:
    enabled: true
  env:
    fileStoreProvider: local
```

Nginx configuration is automatically generated. If Parabol was deployed without Nginx and then Nginx was enabled, there will be a small downtime during the switch.

## S3 Storage

When `FILE_STORE_PROVIDER` is set to `s3`, Parabol will serve assets from S3, but depending on the `PROXY_CDN` variable, **static** assets might be served through Parabol's NodeJS or through S3. **Dynamic** assets are always served through AWS S3.

`CDN_BASE_URL` is the base URL to serve the assets. It might be a load balancer serving the bucket or a CloudFront distribution. It must have the following format: `//my-cdn-domain.com/ENVIRONMENT` where `ENVIRONMENT` is the environment name or a suffix.

### Private S3 bucket

#### AWS Bucket Setup

**1. Create the S3 bucket:**

```bash
aws s3api create-bucket \
  --bucket $BUCKET_NAME \
  --region $BUCKET_REGION
```

For regions other than `us-east-1`, add `--create-bucket-configuration LocationConstraint=$BUCKET_REGION`.

**2. Block all public access:**

```bash
aws s3api put-public-access-block \
  --bucket $BUCKET_NAME \
  --public-access-block-configuration \
    "BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true"
```

**3. IAM permissions required:**

The role/user generating signed URLs needs:

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "uploads",
            "Effect": "Allow",
            "Action": [
                "s3:GetObject",
                "s3:PutObject",
                "s3:DeleteObject"
            ],
            "Resource": "arn:aws:s3:::$BUCKET_NAME/*"
        },
        {
            "Sid": "checkExists",
            "Effect": "Allow",
            "Action": [
                "s3:ListBucket"
            ],
            "Resource": "arn:aws:s3:::$BUCKET_NAME"
        }
    ]
}
```

#### Parabol Configuration

If the S3 bucket is private:
- **static** assets must be served through Parabol, as they are not available directly through S3. This might impact performance if not served through Nginx.
- **dynamic** assets are always served through S3 using presigned urls.

Example of configuration serving static assets through Parabol's NodeJS:

```yaml
parabolDeployment:
  env:
    proxyCDN: true
    fileStoreProvider: s3
    awsAccessKeyId: "key_AWS_ACCESS_KEY_ID"
    awsRegion: "key_AWS_REGION"
    awsSecretAccessKey: "key_AWS_SECRET_ACCESS_KEY"
    awsS3Bucket: "BUCKET_NAME"
    cdnBaseUrl: "//my-cdn-domain.com/test-environment"
```

Configure Parabol with the following parameters in order to serve them through Nginx instead:

```yaml
parabolDeployment:
  nginx:
    enabled: true
  env:
    proxyCDN: true
    fileStoreProvider: s3
    awsAccessKeyId: "key_AWS_ACCESS_KEY_ID"
    awsRegion: "key_AWS_REGION"
    awsSecretAccessKey: "key_AWS_SECRET_ACCESS_KEY"
    awsS3Bucket: "BUCKET_NAME"
    cdnBaseUrl: "//my-cdn-domain.com/test-environment"
```

### Public S3 bucket

If the S3 bucket is public, all assets will be served through S3.

```yaml
parabolDeployment:
  env:
    fileStoreProvider: s3
    awsAccessKeyId: "key_AWS_ACCESS_KEY_ID"
    awsRegion: "key_AWS_REGION"
    awsSecretAccessKey: "key_AWS_SECRET_ACCESS_KEY"
    awsS3Bucket: "BUCKET_NAME"
    cdnBaseUrl: "//my-cdn-domain.com/test-environment"
```
Assets will be served through `https://my-cdn-domain.com/test-environment/build` and `https://my-cdn-domain.com/test-environment/assets`.
